## Novelty Memo

The useful new observation from the `semantics` lens is that ADR-0019 was implemented in code and partially documented at the top of the protocol, but the old word “frontmatter” persisted in operator-facing surfaces below the fold. That is a semantic drift pattern: the system behavior was right, but the signs users read still pointed at the pre-sidecar model.

The useful new observation from the `semantic-relations` lens is that the precise `relations.boundary` warning had been added in the protocol, but summary layers kept compressing it back into the word "handoff." That compression matters because the runtime mechanic is exclusion, not deference, and the wrong word changes how authors write relation reasons.

The useful new observation from the `ontology-modeling` lens is that the repo's RDF/OWL projection has to distinguish operational routing claims from formal class-theory claims everywhere, including comments and ADR update notes. `sg:disjointOwnership` is not `owl:disjointWith`; the difference is the whole ontology-level safety property.

The useful new observation from the `taxonomy-design` lens is that current-vs-retired classification drift is easiest to spot where docs show category tables. A table titled "Current Domain Paths" carried more false authority than scattered prose would have; replacing it with subject-local `taxonomy_domain` guidance makes the organizing principle visible.

The useful new observation from the `knowledge-modeling` lens is that ADR-0019 is not just a storage change; it changes the representation paradigm. `SKILL.md` is now the authored frame, `audit-state.json` is loop-owned state, and `skills.manifest.json` is the compiled joined view. Diagrams that collapse those artifacts back into one node make future docs drift more likely.

The useful new observation from the `context-graph` lens is that stale topology survives longest in diagrams and worked examples. They are compressed graph views, so one old edge name (`adjacent`) or old source node (`schema_version` in frontmatter) can teach the wrong traversal path more forcefully than a paragraph of prose.

The useful new observation from the `context-engineering` lens is that "load the relevant skills" is not enough as an instruction in a repo with many generated, historical, and audit-artifact surfaces. The instruction also needs selection rules: which file owns the fact, which context is current, which surfaces are generated or historical, and when to compact raw search output into a verified evidence list.

The useful new observation from the `skill-infrastructure` lens is that deterministic gates can be correct in `package.json` while active docs still teach an old gate topology. Health-tooling claims need to name both the command wiring and the latest command result; otherwise agents keep treating a repaired gate as intentionally separate debt.

The useful new observation from the `skill-router` lens is that boundary exclusion must happen before co-loading, not after it. Otherwise the router can correctly exclude an adjacent skill and still import that excluded skill's verification partner, reintroducing wrong-domain context through a side door.

The useful new observation from the `skill-scaffold` lens is that the authoring template is more dangerous than ordinary stale prose because it is copied into future skills. A single retired word such as "archetype" in the template can become new CONTENT debt across the library; the scaffold has to model both the current frontmatter fields and the sibling `audit-state.json` split.
